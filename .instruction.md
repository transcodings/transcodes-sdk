# Rollup.js & Changesets 가이드

## 1. Rollup.js란?

Rollup은 **npm 라이브러리 배포에 특화된 번들러**입니다.  
webpack이 앱 번들링에 강하다면, Rollup은 라이브러리 패키지 만들 때 씁니다.

### 핵심 개념

```
src/ (TypeScript 소스)
    ↓ rollup 번들링
lib/
  ├── index.cjs      ← CommonJS (Node.js, require)
  └── index.esm.js   ← ESM (import/export, Tree Shaking)
```

### 기본 설정 (rollup.config.js)

```javascript
import typescript from 'rollup-plugin-typescript2';
import del from 'rollup-plugin-delete';

export default {
  input: 'src/index.ts', // 진입점
  output: [
    {
      file: 'lib/index.cjs',
      format: 'cjs', // require() 방식
      exports: 'named',
    },
    {
      file: 'lib/index.esm.js',
      format: 'esm', // import 방식 (Tree Shaking 지원)
      exports: 'named',
    },
  ],
  plugins: [
    del({ targets: ['lib/*'] }), // 빌드 전 lib/ 초기화
    typescript({ useTsconfigDeclarationDir: true }), // TS → JS 변환
  ],
};
```

### 외부 라이브러리가 있을 때

```javascript
import resolve from '@rollup/plugin-node-resolve';
import commonjs from '@rollup/plugin-commonjs';

const external = [...Object.keys(packageJson.peerDependencies || {})];

export default {
  plugins: [
    resolve(), // node_modules에서 패키지 탐색
    commonjs(), // CJS 패키지를 ESM으로 변환
  ],
  external, // peerDependencies는 번들에 포함 안 함
};
```

| 플러그인                      | 역할                             |
| ----------------------------- | -------------------------------- |
| `rollup-plugin-delete`        | 빌드 전 `lib/` 폴더 삭제         |
| `rollup-plugin-typescript2`   | TypeScript 컴파일 + `.d.ts` 생성 |
| `@rollup/plugin-node-resolve` | `node_modules` 패키지 탐색       |
| `@rollup/plugin-commonjs`     | CJS → ESM 변환                   |

### package.json 설정

```json
{
  "main": "lib/index.cjs",
  "module": "lib/index.esm.js",
  "types": "lib/index.d.ts",
  "sideEffects": false,
  "publishConfig": {
    "access": "public"
  },
  "scripts": {
    "build": "rollup -c",
    "release-package": "changeset publish"
  }
}
```

## 2. Changesets란?

**버전 관리 + CHANGELOG 자동화 + npm 배포 자동화 도구**입니다.

```
수동:
  package.json 버전 수정 → CHANGELOG 작성 → npm publish

Changesets:
  npx changeset → (나머지 자동)
```

---

## 3. 초기 설정 (init)

### 설치

```bash
npm install -D @changesets/cli
```

### 초기화

```bash
npx changeset init
```

생성되는 구조:

```
.changeset/
  └── config.json    ← Changesets 설정 파일
```

### config.json 설정

```json
{
  "$schema": "https://unpkg.com/@changesets/config@3.0.0/schema.json",
  "changelog": "@changesets/cli/changelog",
  "commit": false,
  "fixed": [],
  "linked": [],
  "access": "public",
  "baseBranch": "main",
  "updateInternalDependencies": "patch",
  "ignore": []
}
```

## 4. 일반 배포 흐름

### Step 1: 변경사항 기록

```bash
npx changeset
```

터미널에서 순서대로 선택:

```
1. 패키지 선택 (스페이스바로 선택)
2. 변경 타입 선택:
   - major: 1.0.0 → 2.0.0  (Breaking Change)
   - minor: 1.0.0 → 1.1.0  (새 기능)
   - patch: 1.0.0 → 1.0.1  (버그 수정)
3. 변경 내용 입력
```

`.changeset/` 폴더에 파일이 생성됩니다:

```
.changeset/
  └── fuzzy-bears-dance.md   ← 자동 생성된 파일명
```

### Step 2: 버전 업데이트

```bash
npx changeset version
```

자동으로:

- `package.json` 버전 업데이트
- `CHANGELOG.md` 생성/업데이트
- `.changeset/` 파일 삭제

### Step 3: 빌드 + 배포

```bash
npm run build
npx changeset publish
```

## 5. Pre-release 모드 (alpha / beta)

### Alpha 시작

```bash
npx changeset pre enter alpha
```

이후부터 버전이 다음과 같이 됩니다:

```
1.0.0 → 1.0.1-alpha.0
```

### Alpha 변경사항 기록 + 배포

```bash
# 변경사항 기록
npx changeset

# 버전 업데이트
npx changeset version
# → 1.0.1-alpha.1, 1.0.1-alpha.2 ...

# 빌드 + 배포
npm run build
npx changeset publish
```

### Alpha → Beta 전환

```bash
# alpha 종료
npx changeset pre exit

# beta 시작
npx changeset pre enter beta
```

버전 변화:

```
1.0.1-alpha.2 → 1.0.1-beta.0
```

### Beta → 정식 배포 전환

```bash
# beta 종료
npx changeset pre exit

# 버전 업데이트
npx changeset version
# → 1.0.1 (정식 버전)

# 빌드 + 배포
npm run build
npx changeset publish
```

## 6. 전체 Pre-release 흐름 요약

```
개발 시작
    ↓
npx changeset pre enter alpha   → alpha 모드 진입
    ↓
코드 수정 → npx changeset       → 변경사항 기록
    ↓
npx changeset version           → 1.0.1-alpha.0
    ↓
npx changeset publish           → npm 배포
    ↓
(반복)                          → 1.0.1-alpha.1, alpha.2 ...
    ↓
npx changeset pre exit          → alpha 종료
npx changeset pre enter beta    → beta 모드 진입
    ↓
npx changeset version           → 1.0.1-beta.0
    ↓
npx changeset publish           → npm 배포
    ↓
(반복)                          → 1.0.1-beta.1, beta.2 ...
    ↓
npx changeset pre exit          → beta 종료
    ↓
npx changeset version           → 1.0.1 (정식)
    ↓
npx changeset publish           → 정식 배포 완료
```

## 7. GitHub Actions 자동화

```yaml
name: Release

on:
  push:
    branches:
      - main
      - prerelease

concurrency: ${{ github.workflow }}-${{ github.ref }}

jobs:
  release:
    name: Release
    runs-on: ubuntu-latest
    steps:
      - name: checkout repository code
        uses: actions/checkout@v4
        with:
          fetch-depth: 0
          token: ${{ secrets.GIT_CHANGESET_TOKEN }}

      - name: setup node.js
        uses: actions/setup-node@v4
        with:
          node-version: 22

      - name: install dependencies
        run: npm ci

      - name: build
        run: npm run build

      - name: Create release pull request or publish
        uses: changesets/action@v1
        with:
          publish: npm run release-package
        env:
          GITHUB_TOKEN: ${{ secrets.GIT_CHANGESET_TOKEN }}
          NPM_TOKEN: ${{ secrets.NPM_CHANGESET_TOKEN }}
```

### 동작 방식

```
main 브랜치에 push
    ↓
.changeset/ 파일 있으면
    → "Version Packages" PR 자동 생성

"Version Packages" PR 머지하면
    → 버전 자동 업데이트
    → npm 자동 배포
    → GitHub Release 자동 생성
```
